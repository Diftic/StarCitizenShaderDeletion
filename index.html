What it does

Everything in one guided run

No manual folder hunting. No registry digging. ShaderCacheNuke finds it all and cleans it for you.

🔍

System Analysis

Full scan before anything is touched — cache sizes, running processes, HAGS status, power plan, and available RAM all in one report.

Star Citizen Shader Caches

Detects SC shader folders across every drive and every channel — LIVE, PTU, EPTU, and TECH-PREVIEW. Nothing slips through.

GPU Driver Caches

Clears NVIDIA and AMD driver caches plus the Windows DirectX shader cache — a common culprit for post-patch stuttering.

Detects HAGS and power plan settings with one-click links to the exact Windows settings page. No guessing where to go.

The results page shows pass/fail for every individual operation — bytes freed, files removed, errors flagged — so you know exactly what happened.

Recommended usage

When to run ShaderCacheNuke

After a Star Citizen patch New patches invalidate old shader data. Clearing before your first post-patch launch prevents stale shaders from compiling mid-session.
After a GPU driver update Driver updates change how shaders are compiled. Old cached data from the previous driver can cause glitches or hitching.
Graphical glitches or corruption Flickering geometry, missing textures, or visual corruption are classic signs of stale or corrupt shader cache entries.
Unexplained FPS drops or stuttering If performance has degraded without an obvious cause, a shader cache flush is the first thing to try before deeper investigation.
Before a play session Run the standby memory clear to reclaim RAM from Windows' background cache — especially useful on 16 GB systems.
Developers

Run from source

git clone https://github.com/Diftic/StarCitizenShaderDeletion.git
cd StarCitizenShaderDeletion
python nuke_shaders_gui.py

Requires Python 3.10+ on Windows 10 / 11. Run as administrator for full functionality.

Good to know

Notes & expectations

Longer first launch after cleaning Star Citizen recompiles shaders from scratch on the next launch. This is normal — the initial load will be longer, and performance improves once shaders are rebuilt.
Administrator required Standby memory clearing and boot-time locked file deletion require elevated privileges. The tool requests admin automatically on launch.
Targeted, not destructive ShaderCacheNuke only modifies files within known GPU driver and SC shader cache locations. It does not touch game files, user settings, or system files.
